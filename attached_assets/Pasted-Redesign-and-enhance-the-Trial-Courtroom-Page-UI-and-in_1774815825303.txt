Redesign and enhance the **Trial (Courtroom) Page UI and interaction** to create a **premium legal-tech simulation experience** with a timeline-based debate flow and a controlled final verdict reveal.

---

### 🎯 Goal

Create a **cinematic, timeline-driven courtroom experience** where:

* Debate unfolds step-by-step (NOT all at once)
* Messages appear sequentially with **word-by-word typing animation**
* The system feels like a real legal proceeding
* The verdict is revealed only after user interaction

---

### 🧱 Layout Structure

Use existing layout:

* Floating top navbar
* Fixed sidebar
* Light beige grid background

Main Sections (horizontal layout):

LEFT → Timeline
CENTER → Debate Panel (Sanitized Channel)
RIGHT → Vault / Detokenized Panel

---

### ⏱️ TIMELINE (Left Side)

* Vertical timeline with nodes

* Each node represents a step:

  * Start Debate
  * Defense Argument
  * Prosecution Argument
  * Rebuttal (optional)
  * Final Verdict

* Active step should be visually highlighted (dot/glow)

* Timeline should update dynamically as debate progresses

---

### ⚖️ CENTER PANEL (Debate Area)

Card-style UI:

* Rounded corners
* Soft shadow
* Light background

Each message block:

* Header:

  * Role (DEFENSE LAWYER / PROSECUTION LAWYER)
  * Colored dot (orange for defense, red/blue for prosecution)
  * Label: "SANITIZED INPUT"

* Body:

  * Text appears **word-by-word (typing animation)**

---

### 🔐 RIGHT PANEL (Vault / Detokenization)

* Light card UI
* Displays processed/detokenized version
* Updates alongside debate
* Slightly less prominent than center panel

---

### ▶️ START BUTTON

Add:
"Start Debate"

On click:

* Begin timeline progression
* Trigger sequential rendering of arguments

---

### ⚙️ DEBATE FLOW LOGIC (CRITICAL)

DO NOT render full debate at once.

Flow:

1. User clicks "Start Debate"
2. Show Defense argument → word-by-word typing
3. Wait 1–2 seconds
4. Show Prosecution argument → word-by-word typing
5. Continue for:

   * Rebuttal (optional)

---

### ⌨️ TYPING ANIMATION

* Split text into words
* Render one word at a time (30–50ms delay)
* Should feel like live typing (not instant, not character-by-character)

---

### 🧠 STATE MANAGEMENT

Maintain:

* currentStep (timeline step)
* isDebateStarted (boolean)
* displayedText (typing animation)
* isDebateComplete (boolean)
* showRevealButton (boolean)
* showVerdict (boolean)
